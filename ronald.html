Pasos para subir mis cambios al repositorio de github

Desde el terminal (GitBash de Windows o la terminal integrada de vsCode) y estando en la carpeta del repositorio (ejemplo: en "~/clases/tarea") haremos estos 6 pasos:
  1. Si queremos ver qué archivos tienen modificaciones pendientes por subir, escribimos 
                                
                            git status
  2. Agregamos todos nuestros cambios más recientes al "stage": 
                                
                            git add  .
  3. Si volvemos a pedir el status, veremos que ahora los archivos cambiados están en verde (lo que simboliza que ya están en el "stage".. es decir.. ya listos para ser "empaquetados" en la "cajita" del commit para que se vayan al repositorio 
                                
                            git status
  4. Generamos un "commit" con los cambios que agregamos al "stage" (y le agregamos de una vez un "mensaje" a ese commit, con el parámetro -m) 
                                
                            git commit -m "aprendiendo el uso de los ids"
  5. Ahora subimos ese commit (que por ahora sólo lo tenemos local en la máquina) al repositorio remoto en internet 
                                
                            git push origin main
    6. origin hace referencia a que se trata de la ubicación que tenemos configurada como el "origen" (que apunta al repositorio de github) main se refiere a la rama ("branch") donde estás parado actualmente (lo que te sale dentro del paréntesis arriba en la carpeta)

  6. Si todo sale como esperamos, deberías poder ver ya tus cambios en la página de github, pero además lo puedes confirmar localmente si tiras de nuevo un status, para darte cuenta de que ahora ya no tienes cambios pendientes (ni en rojo ni en verde) 
                                
                            git status
